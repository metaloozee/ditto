- Flue/provisioning output is compacted in places but not consistently redacted before D1/UI persistence.

- `ProjectCoordinator` admits read-only runs, but mutating runs still bypass it and use `projects.activeAgentRunId`.
- Mutating Flue tools do not exist yet; they need coordinator lease checks before every mutation.
