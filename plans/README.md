## Current state after Phase 5 planning

Review date: 2026-07-04. Verified at commit `f09866f` after Phase 4 implementation landed.

Phases 0–4 of `docs/four-layer-flue-workflow-rewrite-prd.md` are substantially implemented in the current app. The audited codebase has the split public `website` Worker plus private `flueWorker`, D1 Flue pointers and snapshot/artifact schema, coordinator SQLite state, read-only `project-coder` route, mutating `ditto-project-run` workflow, lease-fenced mutating tools, stream resume, cancellation/late-event gating, run-end and periodic R2 snapshot checkpoints, snapshot restore preference, restoring pause, restore retry, and checkpoint/restore status in the project UI.

Phase 5 planning found four actionable gaps:

- Real diff artifacts are still missing from the live Flue path. `run_artifacts` and `artifactKey(..., "diff", ...)` exist, but `FlueRunBridge` writes only a text `final_change_summary` and the UI renders `diff_ready` as "Diff is ready." Plans 044 and 045 split this into durable artifact persistence and `@pierre/diffs` UI rendering.
- Explicit export is missing. There is no user-clicked branch/commit/PR action for a reviewed run diff. Plan 046 adds that path with existing GitHub authorization, keeping autonomous agent push/PR behavior out of scope.
- Coordinator leases have no expiry/renewal. The current mutating lease uses a fencing token but has no `expiresAt`, `/renew`, or stale-lease recovery. Plan 047 closes that coordinator responsibility.
